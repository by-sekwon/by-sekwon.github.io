외부 데이터 읽기

텍스트 마이닝의 출발점은 원본 텍스트 데이터를 Python으로 불러오는 것이다. TXT 파일은 웹(URL)에서 직접 읽거나 로컬(내부) 파일로 저장된 것을 불러올 수 있으며, 결과 데이터 구조는 문자열(str), 리스트(list), 데이터프레임(DataFrame) 중 하나로 관리한다.

1. TXT 파일 읽기 → 문자열(str)

urllib.request 모듈을 사용하면 인터넷에 공개된 TXT 파일을 URL로 직접 가져올 수 있다. 한글 텍스트의 경우 decode('utf-8') 옵션이 필수이며, 읽어온 결과는 하나의 긴 문자열로 저장된다.

# 웹페이지 TXT 파일 읽기 → 문자열(str)
import urllib.request
url = ('http://wolfpack.hnu.ac.kr/big_data/data/
%EA%B0%9C%EC%97%AD%EA%B0%9C%EC%A0%95.txt')
request = urllib.request.Request(url)
response = urllib.request.urlopen(request)
# 한글 문장 읽어오기: decode('utf-8') 필수
ds_str = response.read().decode('utf-8')
print(type(ds_str))
print(ds_str[0:40])
<class 'str'>

땅이 혼돈하고 공허하며 흑암이 깊...

💡 \ufeff(BOM: Byte Order Mark)란?

출력 결과 앞에 '\ufeff' 문자가 나타나는 경우, 파일이 UTF-8 with BOM 형식으로 저장된 것이다.

이 문자는 실제 분석 시 노이즈가 되므로 ds_str.lstrip('\ufeff') 또는

decode('utf-8-sig')로 제거한다.

1.2 웹(URL)에서 TXT 파일 읽기 → 데이터프레임(DataFrame)

pandas의 read_fwf() 함수를 사용하면 URL에서 고정 너비 텍스트 파일을 바로 DataFrame으로 읽을 수 있다. 각 행이 DataFrame의 한 행(row)으로 저장되어 이후 문장 단위 분석이 용이하다.

# 웹 TXT 파일 → pandas DataFrame으로 읽기
import pandas as pd
url = ('http://wolfpack.hnu.ac.kr/big_data/data/
%EA%B0%9C%EC%97%AD%EA%B0%9C%EC%A0%95.txt')
# read_fwf: 고정 너비 텍스트 파일 읽기
df = pd.read_fwf(url, encoding='utf-8')
print(type(df))
print(df.head(3))

1.3 로컬 TXT 파일 읽기 → 리스트(list)

open() 함수와 readlines() 메서드를 사용하면 로컬에 저장된 TXT 파일을 줄 단위로 읽어 리스트로 저장할 수 있다. 각 줄의 끝에 줄바꿈 문자('\n')가 포함되므로 분석 전에 strip()으로 제거하는 것이 좋다.

# 로컬(내부) TXT 파일 읽기 → 리스트(list)
# Google Colab 경로 예시
path = '/content/drive/MyDrive/data/bible_genesis.txt'
with open(path, 'r', encoding='utf-8-sig') as file:
genesis = file.readlines() # 줄 단위로 읽어 리스트로 저장
print(type(genesis))
print(genesis[0:3])
<class 'list'>

'하나님이 이르시되 빛이 있으라 하시니 빛이 있었고\n']

📌 encoding='utf-8-sig' vs 'utf-8'

- utf-8 : BOM 문자(\ufeff)가 포함된 채로 읽힌다.

- utf-8-sig: BOM 문자를 자동으로 제거하여 읽는다. 한글 TXT 파일에는 utf-8-sig를 권장한다.

1.4 TXT 읽기 방법 비교

방법 함수 결과 타입 문장 단위 분석 주요 용도
URL → 문자열 urllib.request str 불가 전체 텍스트 처리
URL → DataFrame pd.read_fwf(url) DataFrame 가능 구조화 분석
로컬 → 리스트 open().readlines() list 가능 줄 단위 처리

Chapter 2. PDF 파일 읽기

PDF 파일에서 텍스트를 추출하려면 전용 라이브러리가 필요하다. 대표적으로 PyMuPDF(fitz)와 PyPDF2 두 가지 방법이 있으며, 최신 환경에서는 PyMuPDF가 더 안정적이고 정확한 텍스트 추출을 제공한다.

2.1 방법 1 — PyMuPDF (fitz) 사용 ★ 권장

PyMuPDF는 빠르고 정확한 텍스트 추출 성능을 제공하는 라이브러리이다. 다국어(한글 포함)를 안정적으로 처리하며, 페이지 단위로 순회하여 전체 내용을 하나의 문자열로 합칠 수 있다.

# PyMuPDF 설치
!pip install pymupdf

path = '/content/drive/MyDrive/data/bible_matthew.pdf'
ds_str = ''
with fitz.open(path) as doc:
for page in doc:
ds_str += page.get_text() # 페이지별 텍스트 추출 후 합치기
print(type(ds_str))
print(ds_str[0:50])

2.2 방법 2 — PyPDF2 사용

PyPDF2는 순수 Python으로 작성된 PDF 처리 라이브러리이다. 단, PyPDF2 3.0 이후 버전부터 API가 변경되어 PdfFileReader 대신 PdfReader를 사용해야 한다. 복잡한 레이아웃이나 한글이 포함된 PDF에서는 PyMuPDF에 비해 추출 정확도가 낮을 수 있다.

Chapter 3. 데이터 유형 변환

Python에서 텍스트 데이터는 문자열(str), 리스트(list), 데이터프레임(DataFrame) 세 가지 형태로 관리된다. 분석 목적에 따라 적합한 형태로 상호 변환하는 방법을 숙지해야 한다.

3.1 문자열 → 리스트 변환

PDF에서 추출한 문자열은 줄바꿈('\n')이 없어 splitlines()로 분리되지 않는 경우가 있다. 이때는 문장 경계를 나타내는 특정 패턴 앞에 '\n'을 삽입한 후 분리한다.

① splitlines() 직접 시도 — 분리 안 되는 경우

# PDF 추출 문자열에는 줄바꿈(\n)이 없는 경우가 많다
pdf_lst = pdf_str.splitlines()
print(type(pdf_lst))
print(pdf_lst[0]) # 전체가 하나로 붙어있음
<class 'list'>
# 문장이 분리되지 않고 하나의 긴 문자열로 출력됨
(마1:8)아사는 여호사밧을 낳고 ... (마1:9)웃시야는 요담을 낳고 ...

② 구분자 삽입 후 splitlines() — 해결 방법

# 문장 경계 패턴 '(마' 앞에 줄바꿈 삽입
pdf_str2 = pdf_str.replace('(마', '\n(마')
print(pdf_str2[0:100]) # 줄바꿈이 삽입되었는지 확인
# 줄바꿈 기준으로 리스트 분리
pdf_lst2 = pdf_str2.splitlines()
print(type(pdf_lst2))
print(pdf_lst2[0:3])
'\n(마1:8)아사는 여호사밧을 낳고 여호사밧은 요람을 낳고 요람은 웃시야를 낳고
\n(마1:9)웃시야는...'

'(마1:9)웃시야는 요담을 낳고 요담은 아하스를 낳고 아하스는 히스기야를 낳고']

💡 더 나은 방법: re.split() 활용

import re

pdf_lst2 = re.split(r'(?=\(마)', pdf_str) # '(마' 앞에서 분리 (look-ahead)

pdf_lst2 = [s.strip() for s in pdf_lst2 if s.strip()] # 빈 항목 제거

→ replace + splitlines() 조합보다 정규표현식 방식이 더 정확하고 간결하다.

3.2 리스트 → 문자열 변환

리스트를 다시 하나의 문자열로 합칠 때는 join() 메서드를 사용하는 것이 가장 간결하다. 구분자(separator)를 원하는 형식으로 지정할 수 있다.

방법 A — join() 메서드 (권장)

# join()을 이용한 리스트 → 문자열 변환 (권장)
separator = ','
ds_str2 = separator.join(pdf_lst2)
print(type(ds_str2))
print(ds_str2[0:50])
<class 'str'>
,(마1:8)아사는 여호사밧을 낳고 여호사밧은 요람을 낳고 요람은 웃시야

방법 B — for문 반복 (원본 코드 방식)

# for문을 이용한 리스트 → 문자열 변환
ds_str2 = ''
separator = ','
for idx, val in enumerate(pdf_lst2):
# 마지막 원소 뒤에는 구분자를 붙이지 않음
ds_str2 += val + ('' if idx == len(pdf_lst2) - 1 else separator)
print(type(ds_str2))
print(ds_str2[0:50])
<class 'str'>
,(마1:8)아사는 여호사밧을 낳고 여호사밧은 요람을 낳고 요람은 웃시야

📌 join() vs for문 비교

- join() : 한 줄로 간결, 메모리 효율적, Python 권장 방식

- for문 : 각 원소별 추가 처리가 필요할 때 유용 (조건부 변환 등)

3.3 리스트 → 데이터프레임 변환

형태소 분석 등 문장 단위 처리를 위해 리스트를 pandas DataFrame으로 변환한다. 각 문장이 하나의 행(row)으로 저장되어 필터링, 그룹화, 열 추가 등 다양한 데이터 처리가 가능해진다.

Chapter 4. 전체 데이터 수집 파이프라인 ★ 심화

PDF 파일을 읽어 텍스트를 추출하고, 전처리를 거쳐 형태소 분석에 적합한 DataFrame으로 만드는 전체 흐름을 하나의 파이프라인으로 구성한다.

# ================================================
# PDF → 문자열 → 리스트 → DataFrame 전체 파이프라인
# ================================================
import fitz
import re
import pandas as pd
# STEP 1: PDF 읽기 → 문자열
path = '/content/drive/MyDrive/data/bible_matthew.pdf'
ds_str = ''
with fitz.open(path) as doc:
for page in doc:
ds_str += page.get_text()
print(f'[STEP 1] 문자열 길이: {len(ds_str)}자')
# STEP 2: 문장 경계 삽입 → 리스트 분리
# 방법 A (정규표현식 권장)
pdf_lst = re.split(r'(?=\(마)', ds_str)
pdf_lst = [s.strip() for s in pdf_lst if s.strip()]
print(f'[STEP 2] 문장 수: {len(pdf_lst)}개')
print(f' 첫 문장: {pdf_lst[0]}')
# STEP 3: 리스트 → DataFrame
ds_df = pd.DataFrame(pdf_lst, columns=['verse'], dtype='str')
print(f'[STEP 3] DataFrame 크기: {ds_df.shape}')
print(ds_df.head(3))
